# 测试设计生成策略分析

## 问题背景

当前模板有 **23 个节点，10 个叶子节点**，实际业务场景可能有：
- **50-100 个叶子节点** (复杂表模型)
- **多个来源表** (5-10 个)
- **复杂 Mapping 规则** (50-200 个字段)

这会导致：
1. **输入上下文过长** - RS + TS + Mapping 可能超过 10K tokens
2. **输出内容过长** - 完整测试设计可能超过 LLM 输出限制
3. **生成质量下降** - 内容过多导致 AI 注意力分散

---

## 方案对比

### 方案 A: 一次生成所有测试设计

```
输入：RS + TS + Mapping + 完整模板
  ↓
AI 处理 (一次性生成)
  ↓
输出：完整测试设计 JSON
```

**优点**:
- ✅ 整体一致性好
- ✅ 实现简单
- ✅ 生成速度快 (一次调用)

**缺点**:
- ❌ 上下文限制问题 (输入 + 输出可能超限)
- ❌ 生成质量不稳定 (内容过多易遗漏)
- ❌ 难以调试 (失败需要重新生成)
- ❌ 无法并行处理

**适用场景**:
- 小型项目 (< 20 个测试点)
- 简单的表模型
- LLM 上下文窗口足够大 (32K+)

---

### 方案 B: 按模板节点分块生成

```
输入：RS + TS + Mapping + 模板分支
  ↓
对每个分支/叶子节点:
  AI 处理 (分块生成)
  ↓
合并所有结果
  ↓
输出：完整测试设计 JSON
```

**优点**:
- ✅ 避免上下文超限
- ✅ 生成质量高 (专注每个点)
- ✅ 支持并行处理
- ✅ 易于调试和重试

**缺点**:
- ❌ 实现复杂度高
- ❌ 需要多次调用 LLM
- ❌ 整体一致性需要额外处理

**适用场景**:
- 中大型项目 (> 20 个测试点)
- 复杂表模型
- 需要高质量生成

---

## 推荐方案：混合策略

结合方案 A 和 B 的优势，实现**分层生成 + 智能分块**：

```
Level 0: 测试场景分析
  ↓
Level 1: L0-数据结果检查, L1-配置/调度作业检查
  ↓ (按 Level 1 分支分块)
┌─────────────────────┬─────────────────────┐
│ 分支 1: L0-数据结果检查  │ 分支 2: L1-配置/调度检查 │
│  - 表/视图检查        │  - 作业配置检查        │
│  - 字段检查          │  - 调度参数检查        │
│  (8 个叶子节点)        │  (2 个叶子节点)        │
└─────────────────────┴─────────────────────┘
  ↓ (每个分支内一次生成)
┌─────────────────────┬─────────────────────┐
│ AI 生成分支 1         │ AI 生成分支 2         │
│ Prompt + 局部模板    │ Prompt + 局部模板    │
└─────────────────────┴─────────────────────┘
  ↓
合并结果 → 完整测试设计
```

## 实现策略

### 策略 1: 按 Level 1 分支分块 (推荐)

```python
class ChunkedDesignGenerator:
    """分块生成测试设计"""
    
    def generate(self, rs, ts, mapping) -> TestDesign:
        # 1. 获取模板的 Level 1 分支
        level1_nodes = self.template_loader.get_nodes_by_level(1)
        
        all_results = []
        
        # 2. 对每个 Level 1 分支分别生成
        for branch in level1_nodes:
            # 提取该分支的局部模板
            local_template = self._extract_subtree(branch)
            
            # 生成该分支的测试设计
            branch_result = self._generate_branch(
                branch=branch,
                local_template=local_template,
                rs=rs, ts=ts, mapping=mapping
            )
            all_results.append(branch_result)
        
        # 3. 合并所有分支结果
        return self._merge_results(all_results)
    
    def _generate_branch(self, branch, local_template, rs, ts, mapping):
        """生成单个分支的测试设计"""
        prompt = BRANCH_PROMPT.format(
            branch_title=branch.title,
            branch_structure=json.dumps(local_template),
            rs_content=self._extract_relevant_rs(rs, branch),
            ts_content=self._extract_relevant_ts(ts, branch),
            mapping_content=self._extract_relevant_mapping(mapping, branch)
        )
        
        response = self.llm_client.generate(prompt)
        return self._parse_response(response)
```

**优势**:
- 每个分支独立生成，避免上下文超限
- Level 1 分支通常 2-5 个，调用次数可控
- 每个分支的 Prompt 可以精简 (只包含相关内容)

### 策略 2: 按叶子节点逐一生成

```python
class LeafNodeGenerator:
    """逐叶子节点生成"""
    
    def generate(self, rs, ts, mapping) -> TestDesign:
        leaf_nodes = self.template_loader.get_leaf_nodes()
        
        results = []
        
        # 对每个叶子节点生成测试点
        for leaf in leaf_nodes:
            # 生成该叶子节点的测试内容
            test_point = self._generate_for_leaf(
                leaf_path=leaf.path,
                rs=rs, ts=ts, mapping=mapping
            )
            results.append({
                "path": leaf.path,
                "content": test_point
            })
        
        # 组装完整设计
        return self._assemble_design(results)
    
    def _generate_for_leaf(self, leaf_path, rs, ts, mapping):
        """为单个叶子节点生成测试点"""
        prompt = LEAF_PROMPT.format(
            leaf_title=leaf_path.split('>')[-1],
            leaf_context=self._get_leaf_context(leaf_path),
            # 只提取相关的 RS/TS/Mapping 内容
            relevant_info=self._extract_relevant_info(leaf_path, rs, ts, mapping)
        )
        
        response = self.llm_client.generate(prompt)
        return self._parse_response(response)
```

**优势**:
- 每次生成内容极少，质量最高
- 支持完全并行处理
- 易于缓存和复用

**劣势**:
- 调用次数多 (10-100 次)
- 成本较高
- 整体一致性需要额外处理

### 策略 3: 智能分块 (最佳实践)

根据内容长度动态选择分块策略：

```python
class SmartChunkedGenerator:
    """智能分块生成器"""
    
    def __init__(self, llm_client, template_path, 
                 max_input_tokens=8000,
                 max_output_tokens=4000):
        self.llm_client = llm_client
        self.template_loader = XMindTemplateLoader(template_path)
        self.template_loader.load()
        
        # 配置 token 限制
        self.max_input_tokens = max_input_tokens
        self.max_output_tokens = max_output_tokens
    
    def generate(self, rs, ts, mapping) -> TestDesign:
        # 1. 估算输入内容长度
        input_tokens = self._estimate_tokens(rs + ts + mapping)
        
        # 2. 估算输出内容长度
        output_tokens = self._estimate_output_tokens()
        
        # 3. 根据长度选择策略
        if input_tokens + output_tokens < self.max_input_tokens:
            # 内容较少，一次生成
            print("[INFO] 内容较少，使用一次生成模式")
            return self._generate_all(rs, ts, mapping)
        
        elif self._get_branch_count() <= 5:
            # 分支较少，按分支生成
            print("[INFO] 按 Level 1 分支生成")
            return self._generate_by_branch(rs, ts, mapping)
        
        else:
            # 内容很多，按叶子节点生成
            print("[INFO] 内容很多，按叶子节点生成")
            return self._generate_by_leaf(rs, ts, mapping)
    
    def _estimate_tokens(self, text: str) -> int:
        """估算 token 数 (中文字符约 1.5 tokens/字)"""
        return len(text) * 1.5
    
    def _estimate_output_tokens(self) -> int:
        """估算输出 token 数"""
        leaf_count = len(self.template_loader.get_leaf_nodes())
        # 每个叶子节点约 100-200 tokens
        return leaf_count * 150
```

## 上下文优化技巧

### 1. 内容精简

```python
def _extract_relevant_info(self, leaf_path, rs, ts, mapping):
    """提取与当前测试点相关的信息"""
    
    # 分析叶子节点路径，判断测试类型
    if "主键" in leaf_path:
        # 只提取主键相关的字段
        relevant_fields = [f for f in ts.fields if f.is_primary_key]
        relevant_mapping = [m for m in mapping if m.target_field in relevant_fields]
    elif "度量" in leaf_path:
        # 只提取度量字段
        relevant_fields = [f for f in ts.fields if f.is_measure]
        relevant_mapping = [m for m in mapping if m.rule_type == 'aggregation']
    else:
        # 使用全部内容
        relevant_fields = ts.fields
        relevant_mapping = mapping
    
    return {
        "fields": relevant_fields,
        "mapping": relevant_mapping
    }
```

def _generate_with_context(self, rs, ts, mapping, previous_results=None):
    """增量生成，利用之前的结果"""

### 3. 摘要压缩

```python
def _compress_context(self, rs, ts, mapping):
    """使用 AI 压缩上下文"""
    
    # 先用 AI 生成摘要
    summary_prompt = f"""
    请摘要以下文档，提取与测试设计相关的关键信息:
    
    【RS 文档】{rs[:2000]}...
    【TS 文档】{ts[:2000]}...
    【Mapping】{mapping[:2000]}...
    
    摘要要求:
    1. 目标表名和来源表名
    2. 关键字段 (主键、度量字段)
    3. 重要业务规则
    4. 特殊处理逻辑
    
    摘要限制在 500 字以内。
    """
    
    summary = self.llm_client.generate(summary_prompt)
    return summary
```

## 性能对比

| 策略 | 调用次数 | 生成时间 | 质量 | 成本 |
|------|---------|---------|------|------|
| 一次生成 | 1 次 | 5-10s | 中 | 低 |
| 按分支 (Level 1) | 2-5 次 | 15-30s | 高 | 中 |
| 按叶子节点 | 10-100 次 | 1-5min | 最高 | 高 |
| 智能分块 | 自适应 | 20-40s | 高 | 中 |

---

## 最佳实践建议

1. **默认使用智能分块策略** - 根据内容长度自动选择

2. **提供配置选项** - 允许用户根据项目规模调整

3. **实现内容精简** - 提取相关信息，减少冗余

4. **支持缓存** - 避免重复生成相同分支

5. **并行处理** - 对独立分支并行生成

6. **进度可视化** - 显示生成进度，提升用户体验

---

## 结论

**推荐方案：智能分块 + 按 Level 1 分支生成**

- 平衡了质量和效率
- 适应不同规模的项目
- 实现复杂度适中
- 成本可控

对于当前模板 (10 个叶子节点)，**按 Level 1 分支生成** 是最佳选择。
