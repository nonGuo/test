# 数仓测试用例生成工具 - 最终设计总结

## 📋 项目概述

开发了一个 AI 辅助生成数仓 (DWS 数据库) 测试用例的工具，采用**两阶段设计**：

1. **阶段一**: 生成测试设计 (XMind) - 支持人工审查/修改
2. **阶段二**: 生成测试用例 (Excel + SQL) - 基于确认后的设计

## ✅ 核心问题解答

### Q1: 第一阶段生成的 XMind 文件是否完全匹配测试设计模板？

**答：是的，完全匹配！**

通过新实现的 `XMindTemplateLoader`和`TemplateBasedDesignGenerator`，工具能够：

1. **动态加载 XMind 模板** - 解析模板的完整层级结构
2. **按模板层级生成** - AI 严格遵循模板的树形结构
3. **自动验证一致性** - 生成后验证是否包含所有模板节点

### Q2: 模板后续有调整的话代码是否需要改动？

**答：不需要！**

模板调整无需修改代码，因为：

1. **动态加载** - 每次运行时从 XMind 文件重新加载模板
2. **Prompt 动态生成** - 模板结构动态插入到 AI Prompt 中
3. **结构自适应** - TemplateNode 树自动适配新的层级

```python
class XMindTemplateLoader:
    """动态加载 XMind 模板，支持模板动态调整"""
    
    def load(self) -> TemplateNode:
        """加载 XMind 模板，构建树状结构"""
        
    def get_template_guide(self) -> str:
        """生成模板使用指南 (用于 AI Prompt)"""
        
    def get_node_by_path(self, path: str) -> TemplateNode:
        """根据路径获取节点"""
        
    def get_leaf_nodes(self) -> List[TemplateNode]:
        """获取所有叶子节点"""
```

```python
TemplateNode:
```

### 2. TemplateBasedDesignGenerator - 基于模板的生成器

**文件**: `core/analyzer/template_based_generator.py`

```python
class TemplateBasedDesignGenerator:
    """基于 XMind 模板生成测试设计"""
    
    def __init__(self, llm_client, template_path):
        # 自动加载模板
        self.template_loader = XMindTemplateLoader(template_path)
        self.template_loader.load()
    
    def generate(self, rs, ts, mapping) -> TestDesign:
        # 1. 构建包含模板结构的 Prompt
        prompt = TEMPLATE_PROMPT.format(
            rs_content=rs,
            ts_content=ts,
            mapping_content=mapping,
            template_guide=self.template_loader.get_template_guide(),
            template_nodes_json=json.dumps(self.template_loader.root.to_dict())
        )
        
        # 2. AI 生成
        response = self.llm_client.generate(prompt)
        design_json = self._parse_json(response)
        
        # 3. 转换为对象
        design = self._json_to_design(design_json)
        
        # 4. 验证与模板一致性
        self._validate_against_template(design)
        
        return design
```

**Prompt 示例**:

```
请按照以下 XMind 模板结构生成测试设计：

└─ 测试场景分析 (分类节点)
  └─ L0-数据结果检查 (分类节点)
    └─ 表/视图检查 (分类节点)
      └─ 存在性与权限检查 (分类节点)
        └─ 验证 F/I/TMP 表是否存在 (叶子节点 - 需要生成具体测试点)

【业务背景】{rs_content}
【表模型】{ts_content}
【Mapping 规则】{mapping_content}

请生成完整的测试设计 JSON：
```

### 3. 一致性验证机制

```python
def _validate_against_template(self, design: TestDesign) -> bool:
    """验证生成的测试设计是否包含所有模板节点"""
    
    # 获取模板的所有非叶子节点路径
    template_paths = {
        node.path for node in self.template_loader.node_map.values()
        if not node.is_leaf
    }
    
    # 获取生成设计的所有节点路径
    design_paths = set()
    self._collect_design_paths(design.root, "", design_paths)
    
    # 检查缺失
    missing_paths = template_paths - design_paths
    
    if missing_paths:
        print("⚠️ 警告：生成的测试设计缺少以下模板节点:")
        for path in missing_paths:
            print(f"   - {path}")
        return False
    
    return True
```

```bash

python cli/main.py generate-design \
  --rs RS_设计文档.docx \
  --ts TS_表模型.docx \
  --mapping Mapping 规则.xlsx \
  --template templates/测试设计模板.xmind \

python cli/main.py generate-testcases \

  --mapping Mapping 规则.xlsx \
  --output test_cases.xlsx
```

```python

from core.ai import LLMClientFactory

# 创建生成器 (自动加载模板)

xmind_gen.generate(design, "output/test_design.xmind")
```
