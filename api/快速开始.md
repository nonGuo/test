## 2. 本地部署

### 步骤 1: 安装依赖

```bash
# 安装主项目依赖
pip install -r requirements.txt

# 安装 API 依赖
pip install -r api/requirements.txt
```

### 步骤 2: 配置环境变量

```bash
cd api
cp .env.example .env
```

编辑 `.env` 文件，配置 API Key:

```ini
# 通义千问 API Key (必需)
QWEN_API_KEY=sk-xxxxxxxxxxxxxxxx

# 或者 OpenAI API Key
# OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx

# LLM 配置
LLM_PROVIDER=qwen
LLM_MODEL=qwen-max
```

### 步骤 3: 启动服务

```bash
# 方式 1: 直接运行
python api/run.py

# 方式 2: 使用 Python 模块
python -m api.app
```

### 步骤 4: 验证服务

```bash
# 健康检查
curl http://localhost:5000/api/v1/health

# 查看配置
curl http://localhost:5000/api/v1/config
```

预期响应:
```json
{
  "status": "ok",
  "version": "1.0.0",
  "timestamp": "2026-03-10T12:00:00"
}
```

## 3. 使用 API

### 方式 1: cURL

```bash
curl -X POST http://localhost:5000/api/v1/generate-design \
  -F "rs=@RS 样例.docx" \
  -F "ts=@TS 样例.docx" \
  -F "mapping=@mapping 样例.xlsx" \
  -F "template=@测试设计模板.xmind" \
  -F "strategy=auto"
```

### 方式 2: Python

```python
import requests

files = {
    'rs': open('RS 样例.docx', 'rb'),
    'ts': open('TS 样例.docx', 'rb'),
    'mapping': open('mapping 样例.xlsx', 'rb'),
    'template': open('测试设计模板.xmind', 'rb')
}

response = requests.post(
    'http://localhost:5000/api/v1/generate-design',
    files=files,
    data={'strategy': 'auto'},
    timeout=300
)

result = response.json()
print(result)
```

### 方式 3: 使用客户端封装

```python
from api.example_usage import TestCaseGeneratorClient

client = TestCaseGeneratorClient()

result = client.generate_design(
    rs_path='RS 样例.docx',
    ts_path='TS 样例.docx',
    mapping_path='mapping 样例.xlsx',
    strategy='auto'
)

if result['success']:
    print(f"生成成功！叶子节点数：{result['data']['stats']['leaf_nodes']}")
```

## 7. 常见问题

### Q1: 服务启动失败

**检查点**:
1. 端口是否被占用：`netstat -ano | findstr :5000`
2. 环境变量是否正确配置
3. 依赖是否完整安装

**解决方案**:
```bash
# 更换端口
export API_PORT=8080
python api/run.py
```

### Q2: API Key 错误

**检查点**:
1. `.env` 文件是否存在
2. `QWEN_API_KEY` 是否正确
3. LLM 服务是否可用

**解决方案**:
```bash
# 验证 API Key
curl https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation \
  -H "Authorization: Bearer $QWEN_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"qwen-turbo","input":{"messages":[{"role":"user","content":"hello"}]}}'
```

### Q3: 请求超时

**原因**: 大型项目生成时间较长

**解决方案**:
1. 增加客户端超时设置
2. 使用按分支/按叶子策略
3. 实现异步任务 (待开发)

```python
# 增加超时时间
requests.post(..., timeout=600)  # 10 分钟
```

## 8. 性能调优

### 调整生成策略

```bash
# 小型项目 - 一次生成
export GENERATION_STRATEGY=full

# 中型项目 - 按分支生成
export GENERATION_STRATEGY=by_branch

# 大型项目 - 按叶子生成
export GENERATION_STRATEGY=by_leaf
```

### 调整 Token 限制

```bash
# 增加最大输入 token 数
export MAX_INPUT_TOKENS=16000
```

### 启用并行处理

修改 `api/app.py` 中的生成器配置:

```python
design_generator = DesignGenerator(
    llm_client=llm_client,
    strategy='by_leaf'
)

# 使用并行模式
design = design_generator._generate_by_leaf_parallel(
    rs_content, ts_content, mapping_content,
    max_workers=5
)
```
