# 数仓测试用例生成工具 - 设计方案

## 1. 项目概述

### 1.1 目标
开发一个 AI 辅助生成数仓 (DWS 数据库) 测试用例的工具，实现：
- 基于 RS/TS/Mapping 文档自动生成测试设计
- 基于确认后的测试设计生成完整测试用例
- 支持测试人员人工审查和修改

### 1.2 输入输出

**输入:**
1. RS 设计文档 - 业务背景和测试要点
2. TS 文档 - 表模型设计及变更内容
3. Mapping 文档 - 字段加工规则

**输出:**
1. XMind 格式测试设计 (按模板层级)
2. Excel 格式测试用例 (含 SQL 脚本和预期结果)

## 3. 核心数据模型

### 3.1 TestDesign (测试设计)

```python
TestDesign
├── root: TestNode           # 根节点
├── source_tables: List[str] # 来源表
├── target_table: str        # 目标表
└── mapping_rules: List[Dict] # Mapping 规则

TestNode
├── title: str              # 节点标题
├── children: List[TestNode] # 子节点
├── priority: str           # 优先级 (high/medium/low)
├── description: str        # 测试要点描述
└── tables: List[str]       # 涉及表
```

### 3.2 TestCase (测试用例)

```python
TestCase
├── case_id: str            # 用例编号
├── case_name: str          # 用例名称
├── category: str           # 分类 (功能/性能/...)
├── scene: str              # 测试场景
├── priority: str           # 优先级
├── description: str        # 测试要点
├── tables: List[str]       # 涉及表
├── pre_condition: str      # 前置条件
├── test_steps: str         # 测试步骤 (SQL)
├── expected_result: str    # 预期结果
└── post_condition: str     # 后置条件
```

## 4. XMind 结构设计

### 4.1 模板层级

基于提供的模板 `测试设计模板.xmind`:

```
测试场景分析 (根节点)
├── L0-数据结果检查
│   ├── 表/视图检查
│   │   └── 存在性与权限检查
│   │       ├── 目标存在性检查
│   │       │   └── 验证 F/I/TMP 表是否存在
│   │       └── 权限配置检查
│   │           ├── 验证 query 账号有来源表查询权限
│   │           └── 验证 erl 账号有来源表，目标表增删改查权限
│   └── 字段检查
│       ├── 数据完整性检查
│       │   └── 验证字段级长度精度与 mapping 一致
│       ├── 数据唯一性检查
│       │   ├── 验证直取类字段与源数据一致
│       │   └── 验证赋值类字段与设计一致
│       └── 数据有效性检查
│           └── 字段功能检查
│               ├── 主外键类检查
│               │   ├── 验证主键字段空值率为 0
│               │   └── 验证主键字段唯一
│               └── 度量类检查
│                   └── 验证度量字段 XX 空置率为 XX
└── L1-配置/调度作业检查
```

### 4.2 XMind ↔ 测试用例 映射

| XMind 层级 | 映射到 TestCase 字段 |
|-----------|---------------------|
| L0/L1 | category (分类) |
| 表/视图检查、字段检查 | scene (场景) |
| 存在性与权限检查等 | sub_scene (子场景) |
| 叶子节点 (如"验证主键字段空值率为 0") | 生成具体用例 |
| priority label | priority |
| notes | description |

## 5. AI Prompt 设计

### 5.1 测试设计生成 Prompt

```
你是一位资深的数仓测试专家，请根据以下输入生成测试设计。

【业务背景】{rs_content}
【表模型】{ts_content}
【Mapping 规则】{mapping_content}

请按以下层级输出测试设计 (JSON 格式):
- L0-数据结果检查
  - 表/视图检查
    - 存在性与权限检查
      - 目标存在性检查
      - 权限配置检查
  - 字段检查
    - 数据完整性检查
    - 数据唯一性检查
    - 数据有效性检查
- L1-配置/调度作业检查

生成规则:
1. 必须覆盖：表存在性、字段完整性、主键检查、度量值检查
2. 根据表类型 (新建/变更) 调整测试重点
3. 标注优先级 (high/medium/low)
```

### 5.2 测试用例生成 Prompt

```
你是一位数仓测试专家，请根据测试设计生成具体测试用例。

【测试设计】{design_content}
【表模型信息】
  - 目标表：{target_table}
  - 来源表：{source_tables}
  - 字段列表：{fields}
【Mapping 规则】{mapping_rules}

为每个叶子节点生成测试用例，包含:
- 用例编号、名称、分类、优先级
- SQL 测试脚本
- 预期结果

SQL 生成规则:
1. 表存在性：SELECT COUNT(*) FROM {table}
2. 主键唯一性：SELECT pk, COUNT(*) FROM {table} GROUP BY pk HAVING COUNT(*) > 1
3. 主键非空：SELECT COUNT(*) FROM {table} WHERE pk IS NULL
4. 数据一致性：SELECT a.field, b.field FROM target a JOIN source b ON a.id = b.id WHERE a.field <> b.field
5. 度量值准确性：SELECT SUM(measure) FROM target vs SELECT SUM(source_measure) FROM source
```

## 6. SQL 模板设计

### 6.1 内置 SQL 模板

```sql
-- 1. 表存在性检查
SELECT COUNT(*) AS cnt FROM {{ target_table }};

-- 2. 字段完整性检查
SELECT COLUMN_NAME, DATA_TYPE, NUMERIC_PRECISION
FROM INFORMATION_SCHEMA.COLUMNS 
WHERE TABLE_NAME = '{{ target_table }}';

-- 3. 主键唯一性检查
SELECT {{ pk_field }}, COUNT(*) AS cnt
FROM {{ target_table }}
GROUP BY {{ pk_field }}
HAVING COUNT(*) > 1;

-- 4. 主键非空检查
SELECT COUNT(*) AS null_cnt
FROM {{ target_table }}
WHERE {{ pk_field }} IS NULL;

-- 5. 直取字段一致性检查
SELECT a.{{ target_field }}, b.{{ source_field }}
FROM {{ target_table }} a
JOIN {{ source_table }} b ON a.{{ join_key }} = b.{{ join_key }}
WHERE a.{{ target_field }} <> b.{{ source_field }};

-- 6. 度量值汇总准确性检查
SELECT 
  SUM(a.{{ measure_field }}) AS target_sum,
  b.source_sum,
  ABS(SUM(a.{{ measure_field }}) - b.source_sum) AS diff
FROM {{ target_table }} a
CROSS JOIN (
  SELECT SUM({{ source_measure }}) AS source_sum
  FROM {{ source_table }}
) b;

-- 7. 空值率检查
SELECT 
  COUNT(*) AS total_cnt,
  SUM(CASE WHEN {{ field }} IS NULL THEN 1 ELSE 0 END) AS null_cnt,
  CAST(null_cnt AS FLOAT) / total_cnt AS null_rate
FROM {{ target_table }};

-- 8. 参照完整性检查
SELECT a.{{ fk_field }}, COUNT(*) AS cnt
FROM {{ target_table }} a
LEFT JOIN {{ ref_table }} b ON a.{{ fk_field }} = b.{{ ref_pk_field }}
WHERE b.{{ ref_pk_field }} IS NULL AND a.{{ fk_field }} IS NOT NULL
GROUP BY a.{{ fk_field }};
```

## 7. 接口设计

### 7.1 CLI 命令

```bash
# 阶段一：生成测试设计
python cli/main.py generate-design \
  --rs RS 文档路径 \
  --ts TS 文档路径 \
  --mapping Mapping 文档路径 \
  --output 输出 XMind 路径

# 阶段二：生成测试用例
python cli/main.py generate-testcases \
  --xmind 已确认的 XMind 路径 \
  --ts TS 文档路径 \
  --mapping Mapping 文档路径 \
  --output 输出 Excel 路径

# 辅助命令
python cli/main.py analyze-xmind --xmind XMind 路径  # 分析 XMind 结构
python cli/main.py init  # 初始化配置
```

### 7.2 Python API

```python
from core import (
    DocumentParserFactory,
    AITestDesignGenerator,
    AITestCaseGenerator,
    XMindGenerator,
    XMindAnalyzer,
    TestCaseExporter
)
from core.ai import LLMClientFactory

# 创建工作流
workflow = TestDesignWorkflow(llm_provider="qwen")

# 阶段一
workflow.stage1_generate_design(
    rs_path="RS.docx",
    ts_path="TS.docx", 
    mapping_path="Mapping.xlsx",
    output_path="test_design.xmind"
)

# [人工审查 XMind 后]

# 阶段二
workflow.stage2_generate_testcases(
    xmind_path="test_design_confirmed.xmind",
    ts_path="TS.docx",
    mapping_path="Mapping.xlsx",
    output_path="test_cases.xlsx"
)
```
